# 🧸 AI Memory Gateway · 使用手册

> 给 AI 一个长期记忆。这是一层夹在你和大模型之间的网关：它帮 AI 记住你、记住你们聊过的一切，并随着时间长出温度和节奏。
>
> 本手册讲**怎么用**；**怎么部署**见 [README.md](README.md)。

---

## 一、五分钟跑起来（完全新手版）

1. **拿一个 LLM Key**：去 [openrouter.ai](https://openrouter.ai) 注册 → Keys → 创建一个 Key（`sk-or-v1-…`）→ Credits 充几美元。
2. **Fork + 部署**：Fork 本仓库 → 上 [zeabur.com](https://zeabur.com) 用 GitHub 登录 → New Project → 选你 fork 的仓库（自动识别）→ 同项目里再加一个 **PostgreSQL**。
3. **填变量**（网关服务 → Variables）：`DATABASE_URL`（点 Postgres 复制）、`GATEWAY_SECRET`（自设一串密码）、`API_KEY`（你的 OpenRouter Key）、`API_BASE_URL=https://openrouter.ai/api/v1/chat/completions`、`MEMORY_ENABLED=true`、`MEMORY_MODEL=anthropic/claude-haiku-4.5`。
4. 部署 → 拿到网址 `https://xxx.zeabur.app`。

> 详细图文步骤、所有变量、升级/备份，见 README。

## 二、打开面板

浏览器访问：`https://你的网址/dashboard?gateway_key=你设的GATEWAY_SECRET`

面板是「主页 + 几个房间」的结构（点主页上的塔罗牌进入）：

| 房间 | 作用 |
|---|---|
| 🏠 **主页** | 心跳（AI 当下情绪）、今日电波一句、记忆/回忆/梦的计数 |
| 🧠 **记忆** | 看 / 搜 / 编辑 / 删除所有记忆；底部有「连根删」自助删除 |
| ⏳ **此刻** | 分层视图（AI 此刻读到什么）+ 感受流（feel） |
| 🌱 **根基** | L5 关系里程碑正文 + 里程碑待审候选 + 梦境日记 + AI 人设/档案 |
| 🌸 **回忆墙** | 珍贵的整段回忆（可配照片）+ 里程碑待审候选 |
| ⚙️ **操作间** | 金色控制台：所有功能旋钮一处管 |

---

## 三、第一次设置（必做三步）

1. **写 AI 人设** → 「根基」房 → System Prompt 编辑框，写你想要的 AI 性格/身份/说话方式。保存即生效（热更新，不用重启）。
2. **填「关于对话对象」档案**（可选）→ 同页，写"你是谁、AI 该怎么称呼你"。它和人设**分开存、分开注入**。
3. **接客户端**（Kelivo / ChatBox 等）：API 地址填 `https://你的网址/v1`、API Key 随便填一个、模型填你设的 `DEFAULT_MODEL`。

> 想个性化名字/主页：在设置页或环境变量填 `USER_NAME`（你的名字）、`AI_NAME`（AI 的名字）、`HOME_TITLE`/`SINCE_DATE` 等。**不填 = 通用空白，AI 上下文里不会出现任何名字。**

## 四、控制台旋钮（操作间）—— 逐个说明

进「操作间」（金色控制台）。每个旋钮可点开/关，热更新、重启保留。**大多默认关着，按需打开。**

| 旋钮 | 是什么 | 建议 |
|---|---|---|
| **记忆提取总开关** | 关掉只存对话不提取记忆 | 默认开 |
| **保质感摘要** | 历史压缩时保留情绪/质感，不压成干事实 | 想要"有温度"就开 |
| **滚动摘要封顶** | 摘要区"早期小结 + 最近 N 段"封顶，**不再随聊天无限变长、稀释人设** | 长期聊**建议开**（见第六节）|
| **feel 体温** | AI 每隔几轮留一句"留在心里的感受"，给语气垫底色 | 想要细腻就开 |
| **露骨收敛** | 私密/露骨记忆默认"想得到、不蹦出来"，靠暗号或明显亲密语境才放开 | 按需 |
| **情绪漂移** | 命中的旧记忆会朝当下心情慢慢漂（±0.1/轮，有封顶）| 进阶，敏感 |
| **自然衰减** | 老的/不重要/久未回忆的碎片定期归档（先 dry-run 看、可撤销）| 库大了再开 |
| **主动浮现** | 对话开头偶尔轻轻提起"心里记着的" | 按需 |
| **做梦** | 跨天自动给过去每天补一篇第一人称"梦境日记+卡片" | 想要仪式感就开 |
| **每梦写可检索条目** | 让 AI 能"回忆起"梦的内容 | 配合做梦 |
| **看图** | 把图片透传给模型 + 看完存一条文字描述记忆 | 要发图就开 |

> 数值参数（注入记忆条数、提取间隔、摘要封顶 N/B、漂移步长…）在控制台下方的滑块/输入框里调，都有范围保护。

## 五、日常怎么用（常见操作）

**看 / 搜 / 改记忆** → 「记忆」房：搜索框、分层 Tab（核心/事件/碎片）、语义搜索、单条编辑/删除、批量勾选删除。

**升级里程碑为"永久"** → AI 会把"改变关系结构的大事"检出成**候选**，挂在「根基」房（L5）和「回忆墙」房顶部的待审列表。你点 **升永久** 进永久层、或 **弃**。**机器从不自动升，全你说了算。**

**连根删（自助删一段）** → 「记忆」房底部「🗑️ 连根删」：填起止时间（或勾"到现在"）→ **预览** 看会删多少 → **连根删**。**删前自动备份、可一键撤销；回忆墙永不删。**

**写一条回忆** → 「回忆墙」房 → ＋写一条回忆：标题、正文、心情、可传照片。这是"珍贵的整段记忆"，独立于自动碎片、长期保存。

**看 AI 此刻读到什么** → 「此刻」房分层视图：输入一句话，逐层看 AI 会读到的人设/档案/L5/今日/检索记忆。只读、不影响数据。

## 六、为什么要开"滚动摘要封顶"

聊得越久，"历史摘要"越长，会把**人设**在 AI 眼里的占比稀释掉（AI"有时不太听人设"往往就是被一大段摘要盖住了）。封顶后：摘要区 = **一块"早期小结"（保留所有定义性大事）+ 最近 N 段详细**，永远不超过 N+1 块。

- 卷掉的老内容会**先检出里程碑 → 候选待审**，确认才进永久层，**不丢大事**。
- 不崩缓存：平时省钱照旧，只在很久才"卷"一次。
- 想开：操作间「滚动摘要封顶」打开即可（N 默认 8、B 默认 4，可调）。

## 七、备份与升级

- **备份**：面板「导出备份」下载记忆 JSON，**升级前习惯性存一份**。
- **升级拿作者新版**：你 fork 仓库页 → Sync fork → Update branch → Zeabur 自动重部署；**数据库自动迁移、数据不丢**（人设/记忆都在你自己的库里，不随代码动）。
- **应急**：记忆系统出问题，把 `MEMORY_ENABLED` 改回 `false` 即退回纯转发，不用改代码。

## 八、常见问题

**Q：会不会泄露我的隐私给别人？**
A：代码通用、**数据各在各的库**。你的人设、记忆、对话全在你自己的 PostgreSQL 里，别人 fork 同一份代码也碰不到你的数据。

**Q：露骨/私密的内容安全吗？**
A：常驻的摘要 / "今日浓缩"有后处理保底，**永不写出露骨细节**；私密记忆默认"收着"，靠你设的暗号（`INTIMACY_UNLOCK_KEYS`）或明显亲密语境才放开。

**Q：记忆越来越多会拖慢/烧钱吗？**
A：每轮最多注入有限条数（可调）；开"滚动摘要封顶"后摘要也不再无限涨；"自然衰减"可定期归档老碎片。

**Q：不会写代码能搞吗？**
A：能。照 README 一步步点就行，部署零代码。
